# Анализ автоматизации тестирования веб-приложения "Покупка тура"

## Общая оценка

Проект реализует **сквозную автоматизацию** тестирования веб-приложения с использованием современных практик и инструментов. Покрыты три основных слоя: UI, API, БД.

---

## Сильные стороны

| Категория | Описание |
|---------|--------|
| **Инструменты** | Использованы актуальные и стабильные технологии: `pytest`, `selenium`, `allure-pytest`, `Docker` |
| **Отчётность** | Allure предоставляет детализированные, читаемые отчёты с шагами, скриншотами, логами |
| **Изоляция среды** | `Docker Compose` обеспечивает воспроизводимость окружения, независимость от локальной машины |
| **Покрытие** | Тестирование на всех уровнях: UI, API, БД — позволяет выявлять ошибки на разных слоях |
| **Организация кода** | Логичная структура: разделение тестов, данных, вспомогательных функций |
| **Удобство запуска** | Чёткие инструкции в `README.md`, легко запустить и настроить |

---

## Области для улучшения

| Проблема | Рекомендация                                              |
|--------|-----------------------------------------------------------|
| **Нет виртуального окружения в инструкции** | Добавить создание `venv` в `README.md` при необходимости  |
| **Нет `.gitignore`** | Добавить файл, чтобы не коммитить артефакты               |
| **Нет `requirements.txt` в репозитории** | Убедиться, что файл есть и актуален                       |
| **Нет CI/CD (GitHub Actions, GitLab CI)** | Добавить пайплайн для автоматического запуска тестов при `git push` |
